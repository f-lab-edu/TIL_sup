### 연산자에서 중요한 점

1. ‘/’ 연산시 분모가 0이 되지 않게 연산 전에 0 체크 처리를 해야 한다.
   나눗셈 중 분모가 0일 경우 ***ArithmeticException발생***
2. % 연산으로 할 수 있는 알고리즘 처리 방법
   실무에서는 주기, 순환, 분산을 표현하는 연산자로 사용
    - 짝수 홀수 구분
    - 순한 구조 : 원형 연결 리스트 구현, 라운드 로빈
    - 배수 판별
    - 범위 제한(Clamp 느낌) : ID, 난수, 해시값 범위 제한에 자주 사용
    - 시간 계산
    - 날짜 주기 계산
    - 해시 분산
    - 그룹 분류
3. 단축 평가
   논리 연산자(`&&`, `||`)를 사용할 때, 전체 표현식의 결과를 결정하기 위해 필요한 최소한의 연산만 수행하고, 이미 결과가 확정되면 나머지 부분의 평가를 생략하는 기술
4. 단항 연산자 - 증감
   ++a, —a : 1 증감 후 대입
   a++, a— : 대입 후 1증감
5. 삼항 연산자를 언제 사용하면 좋을까?
   삼항 연산자는 가독성을 높일 수도 있고, 망칠 수 도 있다. 삼항 연산자는 복잡하지 않은 단순한 처리일 때 가독성을 높인다.
- 단순한 기본 값 처리 : `String name = input != null ? input : "guest";`
- 단순한 UI / 출력용 : `String status = isActive ? "ACTIVE" : "INACTIVE";`
- early-return 대체 : `return condition ? SUCCESS : FAIL;`

### 참조 연산자

참조 자료형에서 사용할 수 있는 연산자는 다음과 같다.

1. 타입 관련 연산자
2. 기타 연산자 중 배열 연산자를 제외한 연산
3. ‘+’ 연산자
4. 비교 연산자 중  ‘==’
   == 연산은 equals, toString을 새로 구현해야함. 그렇지 않으면 Object 클래스에서 구현된 toString의 값이 사용돼서 객체의 해시값이 비교됨.
   그래서 String 객체도 ‘==’ 로 비교하면 의미가 없고, equals로 실제 값 비교를 해야 한다.
