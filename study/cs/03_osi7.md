**Open Systems Interconnection Reference Model**

OSI 7계층은 네트워크 통신이 일어나는 과정을 7단계로 나눈 네트워크 표준 모델이다. ISO라는 국제 표준화 기구(ISO, International Organizatinon for Standardzation)에서 정의하였다. 각 계층을 Layer라고 하며 L로 줄여서 부르기도 한다.

> ex) Physical layer (L1), Session layer (L5)
>

![나무위키 - [https://namu.wiki/w/OSI 모형](https://namu.wiki/w/OSI%20%EB%AA%A8%ED%98%95)](attachment:88fad8a7-8b48-482b-a22b-1dee7544ef6d:image.png)

나무위키 - [https://namu.wiki/w/OSI 모형](https://namu.wiki/w/OSI%20%EB%AA%A8%ED%98%95)

네트워크 통신은 7단계로 이루어지며, 송신부에서 응용 계층 ~ 물리 계층순으로 거친다음 수신부에서는 반대로 물리 계층~응용 계층으로 7단계를 수행한다. 우리가 보내는 데이터는 이 계층들을 모두 지나야 무사히 목적지에 도달하고, 다시 돌아오게 되는 것이다. 그렇다면, 왜 이렇게 많은 계층을 건너야할까?

계층을 나누는 이유는 각 계층마다 역할이 있기 때문이다. 이렇게 역할을 모듈화에 독립적으로 구성하게 하면, 전체적인 흐름을 이해하기 쉽고, 문제가 발생하면 어떤 문제인지에 따라 어느 곳에 문제가 생겼는지 쉽게 파악하기 쉽다. 예를 들어, 인터넷 연결 불량시 케이블(1계층) 문제인지, IP 설정(3계층) 문제인지 계층 별로 신속히 진단할 수 있다.

그리고 모듈화로 인해 다른 계층에 영향을 주지 않고 기술 개발을 할 수 있다.

### PDU (Protocol Data Unit)

PDU는 OSI 각 계층에서 다루는 데이터 단위를 말한다. 상위 계층에서 데이터가 내려오면, 각 계층이 헤더를 붙여 자기 계층의 PDU로 만들고(Encapsulation), 수신 측에서는 반대로 캡슐화된 단위로 헤더를 하나씩 확인하고 제거하며(Decapslulation) 올라간다.

**PDU가 필요한 이유**

- 계층마다 역할이 다르기 때문이다.
    - 각 계층은 자기 계층에 필요한 제어정보(주소, 오류검사, 포트 시퀀스 등)을 헤더에 담는다.
- 캡슐화 단위로 하는 이유
    - 디버깅, 성능/오류 추적, 패킷 캡쳐에서 기준이 된다.

계층별 다루는 데이터는 다음과 같다.

| **OSI 계층** | **계층 이름** | **PDU 이름(대표)** | **핵심 예시** |
| --- | --- | --- | --- |
| 7 | 응용 (Application) | Data / Message | HTTP, DNS, SMTP |
| 6 | 표현 (Presentation) | Data | 인코딩(UTF-8), 압축, 암호화(TLS 개념 포함) |
| 5 | 세션 (Session) | Data | 세션 관리(연결 유지/동기화 등) |
| 4 | 전송 (Transport) | **Segment (TCP)** / **Datagram (UDP)** | TCP, UDP |
| 3 | 네트워크 (Network) | **Packet** | IP(IPv4/IPv6), ICMP |
| 2 | 데이터 링크 (Data Link) | **Frame** | Ethernet(802.3), Wi-Fi(802.11) |
| 1 | 물리 (Physical) | **Bits** | 전기/광 신호, 케이블/무선 |

## 계층별 설명

### 응용 계층 (Application Layer - 7계층)

응용 계층은 사용자가 네트워크 자원에 접근하는 방법을 제공한다. 최종적으로 사용자가 볼 수 있는 유일한 계층으로 모든 네트워크 활동의 기반이 되는 인터페이스를 제공한다.

- HTTP, SMTP, SSH, FTP, Telnet, DNS 등이 있다.

### 표현 계층 (Presentation Layer - 6계층)

응용 계층으로부터 전달받은 데이터를 읽을 수 있는 형식으로 변환한다.

- 데이터의 인코딩 디코딩이 이루어진다.
- 데이터의 암호화와 복호화를 수행한다.
- ASCII, MPEG, JPEG, MIDI

### 세션 계층 (Session Layer - 5계층)

두 컴퓨터 간의 세션을 관리하며, Port 연결이라고도 한다. 모든 통신 장치 간에 연결을 설정하고 관리한다. TCP/IP 세션 생성과 통신하는 사용자들을 동기화하고 오류 복구 명령들을 일괄적으로 다루는 계층이다.

- 전이중, 반이중 여부를 확인한다.
- 체크 포인팅과 유휴, 재시작 과정 등을 수행한다.
- 세션 확립, 유지, 중단하는 모든 작업을 수행하는 계층이다.
- NetBIOS, SAP, SDP, PIPO, SSL, TLS

### 전송 계층 (Transport Layer - 4계층)

양 끝단의 사용자들 간의 신뢰성있는 데이터를 주고 받을 수 있도록 해 준다.

> PDU : 세그먼트
>
- 오류검출 및 복구, 흐름제어와 중복검사 수행
    - 시퀀스 넘버 기반의 오류 제어 방식을 사용한다.
- 상위 계층들이 데이터 전달의 유효성이나 효율성을 생각하지 않도록 한다.
- TCP, UDP, SPX, STCP, OSPF, SRD, SCTP
- L4 Switch

### 네트워크 계층 (Network Layer - 3계층)

네트워크 계층은 네트워크 호스트의 IP 주소를 확인하고, 데이터 스트림을 더 작은 단위를 분할하며 때에 따라서 오류를 감지하고 처리한다.

> PDU : 패킷
>
- 여러 개의 노드를 거칠 때마다 경로를 찾아주는 역할을 하는 계층이다.
- 다양한 길이의 데이터를 네트워크를 통해 전달하고, 그 과정에서 전송 계층이 요구하는 서비스 품질을 제공하기 위한 기능적, 절차적 수단을 제공한다.
- 라우팅, 흐름 제어, 세그멘테이션, 오류제어, 인터네트워킹 등을 수행한다.
    - 라우팅이란, 알고리즘에 의해 최대한 빠르게 보낼 최적의 경로를 선택하는 과정이다.
- IP, IPX, IPsec, ICMP, ARP
- 라우터, L3 스위치, VPN 등

### 데이터링크 계층 (DataLink Layer - 2계층)

물리계층을 통해 송수신되는 정보의 오류와 흐름을 관리하여 안전한 정보를 전달한다.

> PDU : Frame
>
- 포인트 투 포인트 간 신뢰성 있는 전송을 보장하기 위해 CRC 기반의 오류 제어와 흐름 제어
- 데이터 재전송 기능
- MAC 주소로 통신
- 브릿지, 스위치, 이더넷
- 데이터 전송 : ASK, PSK, FSK, DM, PCM 등이 사용

### 물리 계층 (Physical Layer - 1계층)

이 계층은 전압, 허브, 네트워크 어댑터, 중계기 및 케이블 사양 등 모든 하드웨어의 물리적 및 전기적 신호를 담당한다.

> PDU : 비트(Bit)
>
- 디지털 → 아날로그 또는 반대로 신호를 변환하는 역할을 한다.
- 전기적인 노이즈의 차단과 혼선 방지, 신호의 손실의 방지와 손실 보상
- 전압, 허브(리피터), 네트워크 어댑터, 중계기 및 케이블
- 10BASE-T, 100BASE-TX, ISDN, wired, wireless

## 계층 별 발생할 수 있는 문제

### L1 물리 계층

- 케이블 불량/단선, 포트 접촉 불량
- 링크 다운, 속도/듀플렉스 불일치
- 무선 신호 약함/간섭

### L2 데이터 링크 계층

- MAC 주소 학습 문제, ARP 이상(ARP 스푸핑/ARP 캐시 꼬임)
- VLAN 설정 오류
- 브로드캐스트 폭주(Loop, STP 문제)

### L3 네트워크 계층

- 라우팅 테이블 오류, 기본 게이트웨이/정저거 라우트 누락
- NAT/방화벽 정책 문제
- MTU 불일치로 인한 단편화/PMTUD 이슈(특정 구간만 끊김)
- DNS는 되는데 IP로만 된다/혹은 반대(경로 문제)

### L4 전송 계층

- TCP 3-way handshake 실패(SYN만 나가고 SYN-ACK 없음)
- 연결은 되는데 느림(재전송, 윈도우 사이즈, 패킷 유실)
- 포트 차단/서비스 미기동(ECONNREFUSED vs timeout)
- UDP 유실/순서 꼬임(실시간 서비스 품질 저하)

**진단 방법**

- 포트 확인
    - 리스닝 : ss -lntp / netstat -lntp
    - 원격 포트 테스트 : nc -vz host port
- TCP 상태 : ss -antp(SYN_SENT, ESTABLISHED, TIME-WAIT 등)
- 재전송/RTT 확인: ss -i, tcpdump + wireshark
- timeout : 경로/방화벽/드롭 등으로 응답 다체가 없음

### L5 세션 계층

- 세션 유지 실패(로그인 실패)
- 로드밸런서 뒤에서 세션 불일치(스티키 세션/세션 저장소 문제)
- WebSocket/장기 연결이 중간에 끊김(idle timeout)

**진단 방법**

- **세션 저장소 확인**: Redis/DB 세션 TTL, 동일 키 갱신 여부
- **LB 설정**: Sticky session, idle timeout, keep-alive 설정
- **애플리케이션 로그**: 세션ID 발급/검증/만료 로직 추적
- **장기 연결**: ping/pong, heartbeat 주기 점검

### L6 표현 계층

- 인코딩 깨짐, JSON 파싱 문제
- 압축 문제로 body 해석 실패
- 암호화/인증서 문제(TLS handshake 실패, 인증서 만료/체인 문제)

**진단 방법**

- **콘텐츠 확인**: Content-Type, charset, Content-Encoding
- **직접 재현**: curl -v, curl --compressed
- **TLS 진단**: openssl s_client -connect host:443 -servername host
- **로그**: “invalid character”, “unexpected end of JSON”, “certificate verify failed”

### L7 응용 계층

- HTTP 4xx/5xx, CORS 에러
- 인증/인가(401/403), 토큰 만료
- API 스펙 불일치(필드명/타입/버전), 페이징/정렬 오류
- DB/외부 API 지연으로 타임아웃(502/504)

**진단 방법**

- **애플리케이션 로그/트레이싱**
    - request id로 end-to-end 추적(APM: Jaeger/Zipkin 등)
- **HTTP 레벨 확인**: curl -v, 헤더/바디/상태코드 분석
- **서버 지표**: latency, error rate, DB 커넥션 풀, GC/CPU/메모리
- **리버스 프록시/LB 로그**: Nginx/ALB 502/504 원인(업스트림 타임아웃 등)

## TCP/IP 4계층

TCP/IP는 OSI 7계층에서 단순화된 모형으로 상업적이고 실무적으로 이용되는 것을 목적으로 만들어졌다. OSI 7계층으로 4~5계층으로 분류하여 적용한다.

응용 계층 - 전송 계층 - 인터넷 계층 - 네트워크 연결 계층이 있다.

### L4 응용 계층 (Application Layer)

사용자와 가장 가까운 계층으로 사용자가 소프트웨어와 소통할 수 있게 해준다. 응용프로그램과 데이터를 교환하기 위해 사용되는 프로토콜이다.

> 파일 전송, 이메일, FTP, HTTP, SSH, Telnet, DNS, SMTP
데이터/메세지
>

### L3 전송 계층 (Transport Layer)

통신 노드 간의 연결 제어 및 자료 송수신 담당. 응용 계층의 세션과 데이터그램 통신서비스를 제공한다.

> TCP, UDP, RTP, RTCP
세그먼트
>

### L2 인터넷 계층 (Internet Layer)

네트워크상 최종 목적지까지 정확하게 연결되도록 연결성을 제공

단말을 구분하기위해 논리적인 주소 IP를 할당

- 출발지와 목적지의 논리적 주소가 담겨있는 IP Datagram이라는 패킷으로 데이터를 변경
- 데이터 전송을 위한 주소 지정
- 라우팅 기능
- 패킷단위의 데이터 구성
    - 세그먼트를 목적지까지 전송하기 위해 시작 주소와 목적지의 논리적 주소를 붙인 단위

> IP, ARP, ICMP, RARP, OSPF
패킷
>

### L1 네트워크 연결 계층 (Network Access Layer/Network Interface Layer)

물리적으로 데이터가 네트워크를 통해 어떻게 전송되는지를 정의한다.

기본적으로 에러검출/패킷의 프레임화를 담당한다.

MAC 주소를 통한 장비간 전송

> MAC, LAN, Ethernet, PPP, Token Ring
