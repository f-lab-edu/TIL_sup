# 소켓(Socket)이란?

소켓(Socket)은 **프로세스(프로그램)가 네트워크를 통해 데이터를 주고받기 위해 OS(커널)가 제공하는 통신 인터페이스(엔드포인트)**다.  
쉽게 말해, 애플리케이션이 네트워크로 통신할 때 사용하는 **“통신용 손잡이”**이고, 대부분의 OS에서는 **파일 디스크립터(File Descriptor)** 로 식별된다.

- 소켓은 표준 유닉스의 **파일 디스크립터 기반 I/O** 모델을 따른다.
    - 그래서 `read/write` 또는 `send/recv` 형태로 다룬다.
- 소켓은 **양방향 통신 채널의 종단점(endpoint)** 이다.
- 소켓은 다양한 주소 체계를 지원할 수 있다.
    - 예: Internet(DARPA) 주소, 로컬 노드 경로 이름(Unix Domain), X.25 주소 등


## 소켓이 하는 일

### 1) 소프트웨어와 소프트웨어를 연결한다
소켓은 **프로토콜 + IP + 포트**를 이용해 통신할 상대(종착지)를 찾아 연결한다.

### 2) 연결된 소프트웨어끼리 데이터를 송수신한다
연결이 성립되면, 양쪽 프로세스는 소켓을 통해 **데이터를 보내고 받고(send/recv)** 통신을 수행한다.


## 클라이언트 소켓과 서버 소켓

- **클라이언트 소켓(Client Socket)**  
  서버에 **연결 요청을 보내는** 소켓

- **서버 소켓(Server Socket)**  
  클라이언트의 **연결 요청을 받는** 소켓
    - `listen`으로 대기하다가 `accept`로 **연결용 소켓**을 새로 얻는다.


## 소켓의 구성 요소(식별 정보)

인터넷 통신에서 소켓은 보통 아래 요소들로 “누구와 통신 중인지”가 정해진다.

- 프로토콜(Protocol) : TCP / UDP
- 로컬 IP 주소(Local IP)
- 로컬 포트(Local Port)
- 원격 IP 주소(Remote IP)
- 원격 포트(Remote Port)

> 특히 TCP에서는 연결이 보통 **(로컬 IP, 로컬 포트, 원격 IP, 원격 포트)** 조합(4-튜플)으로 식별된다.


## 소켓 통신의 작동 원리(전체 흐름)

1. 서버가 소켓을 열고 대기 상태를 만든다.
2. 클라이언트가 서버의 IP/Port로 연결을 요청한다.
3. 서버가 요청을 수락하면(accept) 연결이 성립된다.
4. 연결된 소켓끼리 데이터를 송수신한다.
5. 통신이 끝나면 소켓을 닫아(close) 연결을 종료한다.


## TCP 기반 소켓 흐름(클라이언트/서버)

### Client
1. `socket()` : 소켓 생성
2. `connect(server_ip, server_port)` : 서버로 연결 시도
3. 연결 성공 후 `read()/write()` 또는 `send()/recv()`로 데이터 송수신(반복)
4. `close()` : 연결 종료

> 참고: `accept()`는 서버에서 수행되며, 클라이언트는 `connect()`가 성공하면 연결된 상태가 된다.

### Server
1. `socket()` : 소켓 생성
2. `bind(ip, port)` : 서버 IP/Port에 소켓을 묶음
3. `listen(backlog)` : 연결 요청 대기열 생성
4. `accept()` : 클라이언트 연결 수락(연결용 소켓 반환)
5. `read()/write()` 또는 `send()/recv()`로 데이터 송수신(반복)
6. `close()` : 소켓 종료


## 인터넷 소켓의 두 가지 유형

### 1) 스트림 소켓(Stream Socket) = TCP

스트림 소켓은 **연결 지향**이며 **신뢰성/순서 보장**을 제공한다.

- 데이터는 **바이트 스트림**으로 취급된다.
- **순서 보장**
    - "1, 2"를 보내면 반대쪽에서도 "1, 2" 순으로 도착
- **경계(boundary)가 없다**
    - 메시지 단위가 아니라 “흐르는 바이트”라서 구분 규칙이 필요
    - 예: 길이(prefix) 먼저 보내고, 그 길이만큼 읽기
- **흐름 제어(Flow Control) 내장**
    - 수신 버퍼 초과(오버런)를 방지하는 메커니즘이 포함됨
- 예시
    - telnet, ssh, http 등

#### TCP(Transmission Control Protocol)
- TCP는 패킷이 **오류 없이**, **순서대로**, **빠짐없이** 도착하도록 보장한다.
- IP는 라우팅을 담당하지만, 데이터 무결성/순서 보장을 책임지진 않는다.


### 2) 데이터그램 소켓(Datagram Socket) = UDP

데이터그램 소켓은 **비연결(무연결)** 방식이며, 속도가 빠르고 가볍다.

- 데이터가 도착할 수도 있고, 유실될 수도 있다.
- 순서가 보장되지 않는다.
- 다만 “도착한 패킷 자체”의 데이터는 체크섬 등으로 검증 가능하다.
- 연결을 유지할 필요가 없다
    - 패킷을 만들고 목적지 정보(IP 헤더 등)를 붙여 전송하면 끝
- 사용처
    - TCP를 쓰기 어렵거나, 일부 손실이 허용되고 지연이 더 치명적인 경우
- 예시
    - DHCP, 멀티플레이어 게임, 오디오 스트리밍, 화상 회의 등
